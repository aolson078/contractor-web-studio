# Reverse Engineer What Works

Use this workflow when the user wants evidence-based content patterns instead of guesswork.

## Follow the Six-Step Framework

### 1. Identify the Right Niche Creators

- Find 10-20 creators in the target niche.
- Prefer creators with strong engagement relative to audience size.
- Prefer consistent posting and clear audience overlap.
- Mix established names with rising creators.

### 2. Collect a Large Sample

- Gather 500-1000 posts when possible.
- Capture post text, format, timing, topic, hook, CTA, and engagement metrics.
- Use tools only when allowed; otherwise use smaller manual samples.

### 3. Analyze Patterns

Measure:

- Top 10% posts by engagement rate
- Best-performing formats
- Best-performing times and days
- Best-performing topics

Review:

- Hook types
- Post length
- Emotional triggers
- CTA patterns
- Repeatable structures

### 4. Turn Findings into a Playbook

Document:

- Top hook patterns
- Best format structures
- Best CTA patterns
- Winning topic categories
- Posting-window patterns

Turn each pattern into a reusable template.

### 5. Layer in Brand Voice

Apply these voice rules:

- Write like a smart friend who figured something out.
- Prefer specifics over vague claims.
- Use short sentences and line breaks.
- Let emotion support the lesson when it is authentic.
- Share discoveries instead of lecturing.

Use this contrast:

- Avoid: "Revenue improved a lot."
- Prefer: "Revenue grew 37% in 6 weeks."

- Avoid: "Pricing was difficult."
- Prefer: "I was nervous enough to delay the pricing email for 3 days."

### 6. Convert Attention into Action

Use soft conversion paths first:

- Newsletter signup
- Free resource
- Keyword comment flow
- Profile visit to a clear CTA

Use direct conversion selectively:

- Product mention inside a valuable post
- Link in comments when platform reach penalizes links
- DM CTA when the value exchange is obvious

## Use the Review Checklist

- Identify 10-20 creators.
- Collect a meaningful sample.
- Rank posts by engagement rate.
- Document top hooks.
- Document top formats.
- Document top CTAs.
- Define voice rules.
- Build a reusable template library.
- Track future performance against the playbook.
